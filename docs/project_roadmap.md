# 高 Beta 指數發佈平台：分工清單與開發步驟

本專案旨在建立一個自動化、具備封閉管理機制的金融指數數據平台。

---

## 1. 職責分工 (Division of Labor)

### 👤 管理者 (您)
1.  **各類帳號申請**：依照教學申請 GitHub, Vercel, Supabase, Resend 帳號。
2.  **API 金鑰保存**：安全保存從上述平台獲得的 URL 與 Keys，並將其填入 `.env` 檔案中。
3.  **業務邏輯確認**：決定指數簡介、更新時間、發佈日等靜態資訊。
4.  **系統運維 (Admin)**：
    *   在開發完成後，由您在後台手動創建訪客帳號。
    *   監控後台 Log，確保訪客登入狀況正常。
    *   必要時進行強迫登出操作。

### 🤖 開發助理 (Antigravity AI)
1.  **資料庫架構設計 (Supabase)**：規劃索引表、成分股表、使用者設定表、登入日誌表。
2.  **前端網頁開發 (Next.js)**：打造具備「金融感」的深色模式介面。
3.  **後台管理系統開發**：實現帳號管理、強制登出、數據補件、描述修改等功能。
4.  **自動化腳本開發 (Python)**：
    *   撰寫「雲端運算版」回測與計算程式。
    *   開發「每日自動發信」串接邏輯。
5.  **自動化排程設定 (GitHub Actions)**：設定早上 7:00 (TPE) 定時執行任務。
6.  **部署引導**：協助將 GitHub 的程式碼連結至 Vercel，確保網站上線。

---

## 2. 專案開發六大步驟 (Project Steps)

### 第一階段：環境與金鑰準備 (進行中)
1.  註冊四大雲端服務。
2.  獲取 URL、Anon Key, Service Role Key 與 Resend API Key。
3.  在本地端將這些 Key 填入 `.env` 檔案。

### 第二階段：資料庫設計與會員機制
4.  **Auth 設定**：在 Supabase 關閉「開放註冊」，改為受控的「邀請/手動新增」。
5.  **SQL 指令開發**：我會產出 SQL 指令，請您在 Supabase 後台貼上，建立必要的資料表。
6.  **Storage 設定**：建立存放圖表與訪客頭像的雲端空間。

### 第三階段：數據自動化管線 (Daily Automation)
7.  **程式升級**：將現有的 Python 回測程式碼改寫，讓它改為「寫入資料庫」與「發送圖片」。
8.  **GitHub Actions 配置**：設定排程腳本，保證每天早上 7:00 自動運行。

### 第四階段：訪客端與管理端開發 (The Website)
9.  **登入介面**：實作帳密登入與 Token 驗證。
10. **數據儀表板**：開發 Nasdaq 與台股指數的動態圖表與成分股表格。
11. **後台管理介面**：
    *   使用者列表：查看登入狀態、一鍵強制登出。
    *   稽核日誌 (Audit Logs)：記錄誰在哪個時間登入。
    *   資料編輯器：修改指數簡介、上傳數據。
12. **個人中心**：訪客修改頭像、個人資料、訂閱開關。

### 第五階段：郵件系統整合
13. **信件模板 (Template)**：設計包含漲跌幅、權重表格與圖表的 HTML 郵件。
14. **整合發信**：在每日自動化腳本結尾加入 Resend 發信邏輯。

### 第六階段：上線部署與壓力測試
15. **正式部署**：將所有程式碼推送到 GitHub，由 Vercel 進行自動部署。
16. **聯調測試**：由您本人使用 `admin` 帳號測試創建帳號、登入紀錄、強制登出。
17. **正式跑版**：觀察首個清晨 7:00 的數據更新與發信狀況。

---

> [!TIP]
> 目前我們正處於 **「第一階段：環境與金鑰準備」**。當您取得金鑰後，我們就會進入 **「第二階段」**，建立資料庫並開啟會員制度。
